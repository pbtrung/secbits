# Cloudflare D1 + Workers Migration Plan

Migrate the backend from Firebase (Firestore + Auth) to Cloudflare D1 (SQLite) + Workers. All client-side crypto is unchanged. The Worker becomes the sole trust boundary between the browser and the database.

## Architecture

```text
Before:  Browser â†’ Firebase Auth + Firestore (Google)

After:   Browser (CF Pages) â†’ Worker (auth + CRUD API) â†’ D1 (SQLite)
```

The browser never touches D1 directly. The Worker enforces ownership: every `/users/:userId/*` route verifies that the JWT `sub` equals `:userId` before executing any query.

## D1 Schema

```sql
CREATE TABLE IF NOT EXISTS users (
  id              TEXT PRIMARY KEY,
  email           TEXT UNIQUE NOT NULL,
  password_hash   TEXT NOT NULL,          -- "saltB64:hashB64" (PBKDF2-SHA256)
  username        TEXT NOT NULL DEFAULT '',
  user_master_key BLOB,                   -- 192 bytes; NULL until first login
  created_at      TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS entries (
  id         TEXT PRIMARY KEY,
  user_id    TEXT NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  entry_key  BLOB NOT NULL,               -- ~192 bytes wrapped key
  value      BLOB NOT NULL,               -- encrypted history, < 999999 bytes
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX IF NOT EXISTS idx_entries_user_id ON entries(user_id);
```

## Worker API

| Method | Path | Purpose |
|--------|------|---------|
| `POST` | `/auth/login` | Verify email + password â†’ issue JWT |
| `GET` | `/users/:userId/profile` | Fetch username + user_master_key |
| `POST` | `/users/:userId/profile` | Save user_master_key (first login) |
| `GET` | `/users/:userId/entries` | List all raw entries |
| `POST` | `/users/:userId/entries` | Create entry |
| `PUT` | `/users/:userId/entries/:entryId` | Update entry |
| `DELETE` | `/users/:userId/entries/:entryId` | Delete entry |

Binary fields (`entry_key`, `value`, `user_master_key`) are base64-encoded in JSON over the wire and stored as BLOB in D1.

## Auth Design

**Password hashing:** PBKDF2-SHA256 (100,000 iterations, 32-byte random salt) via the Web Crypto API. No npm dependency â€” available natively in both Workers and Node.js 18+. Stored as `"saltB64:hashB64"` in the `password_hash` column.

**JWT:** HS256 signed with a `JWT_SECRET` Worker secret (set via `wrangler secret put JWT_SECRET`). Payload: `{ sub: userId, exp: now + 8h }`. Implemented via Web Crypto HMAC-SHA256 â€” no external JWT library. Token stored in a module-level variable in `api.js` and sent as `Authorization: Bearer <token>` on every request.

**User creation:** No registration endpoint. Users are inserted into D1 via CLI using `scripts/create-user.mjs`:

```bash
node scripts/create-user.mjs --email user@example.com --password secret --username Alice
# outputs a wrangler d1 execute command ready to run
```

## Config File Format

Replaces the Firebase `auth` block and `db_name` field:

```json
{
  "worker_url": "https://secbits-api.<account>.workers.dev",
  "email": "user@example.com",
  "password": "your-login-password",
  "root_master_key": "<base64-encoded key, >=256 bytes when decoded>"
}
```

## Client-Side Changes

`src/firebase.js` is replaced by `src/api.js`. All exported function names are preserved so no component changes are needed. Internal implementation changes:

| Function | Change |
|----------|--------|
| `initFirebase(config)` | Renamed `initApi(config)`. Stores `worker_url`, calls `POST /auth/login`, stores JWT. |
| `signIn(email, password)` | Inlined into `initApi`; no longer a public export. |
| `fetchUser(userId)` | `GET /users/:userId/profile` |
| `saveUserMasterKey(userId, blob)` | `POST /users/:userId/profile` |
| `fetchRawUserDocs(userId)` | `GET /users/:userId/entries`; decodes base64 â†’ `Uint8Array` for binary fields. |
| `fetchUserEntries(userId)` | Calls new `fetchRawUserDocs`; crypto logic unchanged. |
| `createUserEntry(userId, entry)` | `POST /users/:userId/entries`; encodes blobs as base64. |
| `updateUserEntry(userId, entryId, entry)` | `PUT /users/:userId/entries/:entryId` |
| `deleteUserEntry(userId, entryId)` | `DELETE /users/:userId/entries/:entryId` |
| `restoreEntryVersion(userId, entryId, hash)` | Fetches raw entry via HTTP, processes locally, writes via PUT. |
| `initUserDataCollection` | **No-op** â€” D1 needs no placeholder document. |
| `setUserMasterKey` / `getUserMasterKey` / `clearUserMasterKey` | **Unchanged** â€” pure in-memory. |
| `__historyFormatTestOnly` | **Unchanged** â€” pure logic, tests unaffected. |

## Verification

```bash
# 1. Create D1 database
cd worker && wrangler d1 create secbits-db

# 2. Apply schema
wrangler d1 execute secbits-db --file schema.sql

# 3. Create a user
node scripts/create-user.mjs --email test@example.com --password secret --username Test
# pipe the output to wrangler d1 execute

# 4. Set JWT signing secret
wrangler secret put JWT_SECRET

# 5. Run Worker locally
wrangler dev

# 6. Run frontend against local Worker
npm run dev  # upload new-format config JSON at login

# 7. Run test suite (only import paths changed in 2 test files)
npx vitest run

# 8. Deploy
wrangler deploy
wrangler pages deploy dist
```
